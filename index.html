GitHub Actions와 노션 API를 활용해 매월 말일에 운영현황 데이터를 자동으로 아카이브 페이지에 누적 저장하는 방법

🔧 난이도 초급
시작 방법
클로드 채팅 질문 예시

"노션의 특정 페이지(운영현황 페이지)의 내용을 매월 말일에 자동으로 기록되도록 하는 방법(해당 페이지 캡쳐 등)을 구현하고 싶어. 표 내용은 relation, rollup 기능을 통해 계산된 숫자들이고, 기록되는 방법은 노션에 저장되는 방법 등이 있을 것 같아. 가장 효율적으로 구현 가능한 방법을 알고 싶어"

단계별 가이드
설정 방법
01

노션 Integration 만들기

노션 API 접근 권한을 위한 토큰을 발급받아요.

  • notion.so/my-integrations 접속
  • "+ New integration" 클릭 → 이름 입력 (예: monthly-capture)
  • 생성 후 "Internal Integration Secret" 옆 Show 클릭 → 토큰 복사
💡 토큰은 ntn_xxx... 형태예요. 외부에 노출되지 않도록 주의하세요.
02

노션 페이지에 Integration 연결

운영현황 페이지의 상위 페이지에 연결하면 하위 페이지 전체에 자동으로 상속돼요.

  • 노션에서 상위 페이지 열기 (예: 등록환자 DB)
  • 우상단 ··· → Connections → 생성한 Integration 클릭
  • 아카이브 페이지에도 동일하게 연결
⚠️ Integration이 연결되지 않으면 API로 페이지를 읽을 수 없어요. 반드시 두 페이지 모두 연결하세요.
03

페이지 ID 복사

두 페이지의 ID를 각각 복사해 두세요.

# URL에서 마지막 32자리가 페이지 ID
https://www.notion.so/3273a4aad721806e99bdf5ae28c741b3
                       ↑ 이 부분만 복사
04

GitHub 저장소 만들기

코드와 자동화 설정 파일을 저장할 Private 저장소를 만들어요.

  • github.com 로그인 → 우상단 "+" → New repository
  • 이름: notion-monthly-capture
  • Private 선택 후 Create repository 클릭
05

저장소 클론하기

생성한 GitHub 저장소를 내 컴퓨터로 복사해요. 이후 로컬에서 파일을 만들고 수정할 수 있어요.

# GitHub 저장소 주소 복사 후 클론
git clone https://github.com/사용자명/notion-monthly-capture.git

# 클론된 폴더로 이동
cd notion-monthly-capture
💡 저장소 주소는 GitHub 저장소 페이지에서 Code 버튼 클릭 → HTTPS 탭에서 복사할 수 있어요.
06

브랜치 만들고 작업하기

변경사항을 main 브랜치에 직접 올리지 않고, 별도 브랜치에서 작업한 뒤 병합하는 방식으로 안전하게 관리할 수 있어요.

# 새 브랜치 생성 및 이동
git checkout -b feature/add-capture-script

# 파일 추가 후 스테이징
git add capture.py
git add .github/workflows/capture.yml

# 커밋
git commit -m "Add notion capture script and workflow"

# 원격 저장소로 푸시
git push -u origin feature/add-capture-script
  • git checkout -b 브랜치명 — 새 브랜치를 만들고 바로 이동
  • git add — 변경된 파일을 커밋 준비 상태로 올림
  • git commit -m "메시지" — 변경 내용을 저장소에 기록
  • git push -u origin 브랜치명 — 로컬 브랜치를 원격에 올림
⚠️ GitHub에 push한 뒤 저장소 페이지에서 Compare & pull request 버튼으로 main 브랜치에 병합할 수 있어요.
07

두 파일 이해하고 준비하기

업로드할 파일은 총 2개예요. 각 파일의 역할을 이해하고 준비해주세요.

capture.py Python 스크립트

노션 API로 운영현황 페이지의 표 데이터를 읽고, 아카이브 페이지에 저장하는 핵심 코드예요.

  • 운영현황 페이지에서 데이터베이스(표)를 자동으로 탐색
  • Relation, Rollup, Formula 등 계산값 포함 전체 데이터 읽기
  • 지정된 열 순서대로 아카이브 페이지에 표 형태로 저장
  • Rollup 미완료 시 최대 3회 재시도 후 저장

열 순서 지정 부분 — 표 열이 바뀔 경우 이 부분만 수정

COLUMN_ORDER = {
    "총 현황":  ["Name", "병원1", "병원2", "합계", "병원1 1"],
    "총 현황2": ["Name", "남성-병원1", "남성-병원2", "Formula"],
}
capture.yml GitHub Actions 설정

GitHub 서버에서 언제, 어떻게 capture.py를 실행할지 정의하는 자동화 설정 파일이에요.

  • 매월 28~31일 오후 6시 (KST) 자동 실행 예약
  • 실행 당일이 진짜 말일인지 체크 후 실행 여부 결정
  • GitHub Secrets에서 토큰과 페이지 ID를 안전하게 불러와 사용
  • 수동 실행 버튼(Run workflow)도 포함

실행 시각 설정 — 변경이 필요한 경우 이 부분만 수정

- cron: '0 9 28-31 * *'
# UTC 09:00 = 한국시간 오후 6시

08

파일 업로드

두 개의 파일을 저장소에 업로드해요.

  • capture.py → 저장소 루트에 업로드
  • capture.yml → .github/workflows/capture.yml 경로로 업로드
💡 폴더 만들기: Add file → Create new file 클릭 후 파일명 입력란에 .github/workflows/capture.yml 을 그대로 입력하면 폴더가 자동 생성돼요.
09

GitHub Secrets 등록

민감한 정보(토큰, 페이지 ID)는 코드에 직접 입력하지 않고 Secrets에 안전하게 저장해요. 코드가 공개되더라도 이 값들은 절대 노출되지 않아요.

저장소 → Settings → Secrets and variables → Actions → New repository secret

Secret 이름 값
NOTION_TOKEN Integration 토큰 (ntn_xxx...)
SOURCE_PAGE_ID 운영현황 페이지 ID (32자리)
ARCHIVE_PAGE_ID 아카이브 페이지 ID (32자리)
⚠️ ID 복사 시 앞뒤 공백이나 줄바꿈이 포함되지 않도록 주의하세요. 오류의 주요 원인이에요.
10

수동 테스트 실행

말일을 기다리지 않고 지금 바로 테스트해볼 수 있어요.

  • 저장소 → Actions 탭 클릭
  • 좌측 "노션 운영현황 월말 자동 기록" 클릭
  • 우측 "Run workflow" 버튼 클릭
  • 초록 체크 ✅ 확인 후 노션 아카이브 페이지에서 결과 확인

FAQ
PC가 꺼져 있어도 정말 작동하나요? ↓
네, 완전히 무관해요. GitHub 서버가 자동으로 실행하기 때문에 내 PC 전원, 노션 로그인 여부, 인터넷 연결 상태와 관계없이 작동해요.
데이터베이스가 늘어나도 코드를 수정해야 하나요? ↓
아니요. 코드가 페이지 안의 데이터베이스를 자동으로 모두 찾아서 처리해요. 데이터베이스가 추가되거나 이름이 바뀌어도 별도 수정 없이 자동으로 반영돼요.
Rollup, Relation 값도 저장되나요? ↓
네. Rollup의 합계/평균 등 계산된 최종값이 그대로 저장돼요. 수식(Formula) 결과값도 마찬가지예요. 단, 연결 구조 자체가 아닌 계산된 숫자 결과만 저장돼요.
비용이 드나요? ↓
완전 무료예요. GitHub Actions 무료 플랜은 월 2,000분을 제공하고, 이 작업은 1회 실행에 약 10~20초만 사용해요. 노션 API도 무료 플랜에서 충분히 사용 가능해요.
노션 토큰이 만료되면 어떻게 되나요? ↓
Internal Integration 토큰은 만료되지 않아요. 단, 토큰을 Refresh하거나 Integration을 삭제하면 GitHub Secrets에 새 토큰으로 업데이트해야 해요.
